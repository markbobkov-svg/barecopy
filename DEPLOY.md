# Barecopy — ранбук деплоя (~40 минут твоего времени)

Пакет: `index.html` (приложение + лендинг + Pro), `privacy.html`, `terms.html`,
`robots.txt`, `sitemap.xml`, `favicon.svg`, `og.png`, `vercel.json`.
Тесты: 86/86 (логика 50 + целостность файлов 27 + Pro-слой 9).

⚠️ Никогда и никому (включая меня) не пересылай ключи, токены и пароли.
Всё секретное вводится только тобой в дашбордах сервисов.

---

## Шаг 1 — Домен (5 мин, €9.65/год)
1. Купи `barecopy.com` (Cloudflare Registrar / Porkbun / Namecheap — где смотрел цену).
2. Только купить. DNS настроим на шаге 3. Домен уже вшит в файлы пакета.

## Шаг 2 — GitHub (5 мин)
1. github.com/new → имя `barecopy`, Public или Private — неважно → Create.
2. «uploading an existing file» → перетащи ВСЁ содержимое zip (не сам zip) → Commit.

## Шаг 3 — Vercel (5 мин)
1. vercel.com → Sign up with GitHub.
2. Add New → Project → Import `barecopy` → Deploy (настройки не трогать —
   это статический сайт).
3. Получишь адрес вида `barecopy.vercel.app` — уже можно открыть и потыкать.
4. Project → Settings → Domains → добавь купленный домен → Vercel покажет
   1–2 DNS-записи → внеси их у регистратора. Подхватится за минуты, максимум пару часов.

## Шаг 4 — Polar (15 мин) — деньги
Модель изменилась: доступ Pro выдаётся ПО ФАКТУ ОПЛАТЫ (сервер проверяет
подписку в Polar через API), а не по ключу активации. Оплата идёт во встроенной
форме прямо на странице, а не в новой вкладке.

1. polar.sh → Sign in with GitHub → Create organization (имя: Barecopy).
2. Products → New Product:
   - Name: `Barecopy Pro`
   - Pricing: Subscription → Monthly, **€4/мес** → Save.
   - License Keys больше НЕ нужны — их можно не добавлять.
3. Скопируй **Product ID** со страницы продукта — это `POLAR_PRODUCT_ID`.
4. Settings → Developers → **New Access Token** (доступ на чтение продуктов /
   подписок / заказов / клиентов и создание checkout + customer-session) →
   скопируй `polar_oat_...`. Это `POLAR_ACCESS_TOKEN` — СЕКРЕТ, живёт только на
   сервере, в браузер не попадает.
5. Vercel → Project → Settings → Environment Variables (Production + Preview),
   добавь:
   - `POLAR_ACCESS_TOKEN` = токен из п.4
   - `POLAR_PRODUCT_ID`   = ID из п.3
   - `JWT_SECRET`         = длинная случайная строка (`openssl rand -hex 32`) —
                            ею подписывается кука сессии; смена = разлогинит всех
   - `APP_ORIGIN`         = `https://www.barecopy.com` (твой прод-домен)
   - `SMTP_*` / `CONTACT_FROM` — те же, что для контактной формы: теперь ими же
     уходит письмо со ссылкой для входа (magic link).
   Save → Redeploy. Полный список — в `.env.example`.
6. config.js трогать НЕ обязательно. `BARECOPY_CHECKOUT` (ссылка `buy.polar.sh/...`)
   используется теперь ТОЛЬКО как запасной вариант, если `/api/checkout`
   недоступен; `BARECOPY_ORG_ID` больше не задействован, но пусть лежит.
7. Для выплат Polar попросит онбординг Stripe: паспорт + IBAN.
   Финляндия поддерживается. Цепочка денег: покупатель → Polar (merchant of
   record, EU VAT — их проблема) → выплата на твой счёт.

## Шаг 5 — Смоук на проде (5 мин)
1. Нажми **Get Pro** → откроется встроенная форма оплаты (оверлей, без ухода со
   страницы). Оплати (потом отмени + refund из дашборда Polar).
2. После оплаты Pro включается САМ, прямо на странице — ключ вводить не нужно,
   загруженный файл не теряется. В шапке появляется «Pro active».
3. Открой сайт в другом браузере → «Already Pro? Sign in with your email» →
   впиши тот же e-mail → на почту придёт ссылка (действует 15 мин) → перейди →
   Pro включён. (Это замена «повторной активации ключа».)
4. «Manage subscription» → должен открыть портал Polar (платёжка / отмена).
5. Накидай 4+ файлов → ZIP + report → открой отчёт.
6. Проверь отмену: после cancel Pro держится до конца оплаченного периода
   (кука до 30 дней; при следующем заходе `/api/me` — источник истины Polar).
7. Если оплата/вход выдаёт ошибку — пришли мне ТЕКСТ ошибки и на каком шаге
   (НЕ токен и НЕ куку): правится в `api/*.js` / `lib/*.js`.

## Обновление sample-report при смене версии
Файлы `sample-report.pdf` и `sample-report-preview.png` — статичные бинарники;
версия печатается ВНУТРИ них и не обновляется сама. После bump'а
`BC_CONFIG.version` (и перед push такой правки) перегенерируй их:

    node tools/gen-sample-report.mjs

Версия читается из index.html, так что всегда «последняя». PDF собирается на
вендоренном pdf-lib (без внешних запросов). Для превью-PNG нужен разовый
рендерер (в репозиторий не кладём):

    npm i --no-save pdf-to-img @napi-rs/canvas && node tools/gen-sample-report.mjs

(node_modules уже в .gitignore.) Данные-образец зашиты в скрипт; при правках
`buildReportPDF` в index.html синхронизируй их с телом функции в скрипте.

**Автоматическая защита (pre-push hook).** Хук блокирует `git push`, если версия
внутри `sample-report.pdf` не совпадает с `BC_CONFIG.version`. Включить один раз
на каждый клон:

    git config core.hooksPath tools/hooks

Хук (`tools/hooks/pre-push` + `check-sample-version.mjs`) работает на голом Node,
без зависимостей. Если сработал — перегенерируй sample по инструкции выше,
закоммить и пушь заново.
